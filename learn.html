100 Essential Git Commands with Explanations

This guide will walk you through the most important Git commands, step by step, with clear explanations.

1. Initializing and Cloning

1. git init

Initializes a new Git repository in the current directory. This creates a .git folder containing all version control information.

2. git clone [url]

Clones an existing repository from a remote source (e.g., GitHub) to your local machine.

2. Staging and Committing

3. git add [file]

Stages a specific file for commit, marking it to be included in the next snapshot.

4. git add .

Stages all changes (new, modified, deleted files) in the current directory.

5. git status

Shows the current status of the repository, including staged, unstaged, and untracked files.

6. git commit -m "message"

Commits the staged changes to the repository with a descriptive message.

3. Branching and Merging

7. git branch

Lists all branches in the repository.

8. git branch [branch-name]

Creates a new branch with the specified name.

9. git checkout [branch-name]

Switches to the specified branch.

10. git merge [branch]

Merges the specified branch into the current branch.

4. Working with Remote Repositories

11. git remote -v

Lists the remote repositories linked to your project.

12. git push

Pushes committed changes to the remote repository.

13. git pull

Fetches and integrates changes from the remote repository into the local branch.

5. Resetting and Reverting

14. git reset --hard

Resets the working directory and staging area to the last commit, discarding changes.

15. git revert [commit]

Creates a new commit that undoes a previous commit.

6. Inspecting Repository

16. git log

Displays the commit history.

17. git diff

Shows the differences between your working directory and the last commit.

More Advanced Commands

The above commands cover the basics of Git. More advanced commands like git cherry-pick, git rebase, and git bisect will be covered in further sections.
