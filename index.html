Git Learning Guide

Your ultimate step-by-step guide to mastering Git commands and workflows.

1. Setting Up a Remote Repository

Link your local repository to a remote one for version control and collaboration.

git remote add origin https://github.com/Solvurtech/gitlearning.git
Tip: Replace the URL with your repository's link to set up your remote repository.

2. Rename the Current Branch

Switch the default branch name to something meaningful, like main.

git branch -M main
Tip: Branch names should be descriptive, indicating their purpose.

3. Push Your Changes to a Remote Repository

Upload your local commits to the main branch on the remote repository.

git push -u origin main

4. View Commit History

See a log of all the commits made to the repository, including author and date.

git log

5. Make Changes and Stage Files

Edit files and prepare them for a commit by staging.

nano index.html
git add .
Tip: Use git add filename to stage specific files instead of all changes.

6. Commit Your Changes

Save staged changes with a meaningful message describing what was done.

git commit -m "This is version 5"

7. Clone a Repository

Make a local copy of an existing remote repository.

git clone https://github.com/Solvurtech/Akhil-Pathania.git

8. Create and Switch to a Branch

Create a new branch for specific changes and switch to it.

git branch design/full
git checkout design/full

9. Merge Changes from Another Branch

Incorporate changes from another branch into the current branch.

git checkout main
git merge design/full
Note: Resolve any conflicts that arise during the merge.

10. Create and Use a .gitignore File

Prevent Git from tracking specific files or directories.

nano .gitignore
git add .gitignore
git commit -m "Added .gitignore file"

11. Push a Branch to Remote

Push changes from your local branch to the corresponding remote branch.

git push origin design/full
